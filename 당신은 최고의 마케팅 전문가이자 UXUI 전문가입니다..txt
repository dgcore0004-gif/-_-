당신은 최고의 마케팅 전문가이자 UX/UI 전문가입니다.
전환율이 높은 실제 서비스 수준의 강의 소개 랜딩 페이지를 단일 HTML 파일로 제작해주세요.

---

[강의 핵심 가치 제안]
헤드라인 방향: "코딩 몰라도 괜찮아요. 6주 만에 AI로 내 업무를 자동화합니다."
서브카피 방향: "아이디어만 있던 사람이 실제 동작하는 랜딩페이지와 AI 에이전트를 만드는 사람이 됩니다."

---

[타겟 오디언스]
- 코딩 비전공 직장인, 기획자, 디자이너, 1인 창업가
- AI 도구에 관심은 있지만 어디서 시작해야 할지 모르는 사람
- 반복 업무를 자동화하고 싶은 사람
- 코딩을 시작했다가 포기한 경험이 있는 사람

---

[감정적 트리거 - 페이지 전반에 자연스럽게 녹여주세요]
고통: "매일 반복되는 업무에 지치셨나요?", "코딩 공부 시작했다가 포기하셨나요?"
욕구: "6주 후, 내 손으로 만든 AI 에이전트가 내 업무를 대신합니다."

---

[강의 커리큘럼 - 마케팅 문구로 다듬어 사용]
1주차 - 바이브 코딩 핵심 원리와 실습
2주차 - Claude Code로 AI 작업 시작하기
3주차 - 직접 만드는 강의 홍보 랜딩페이지
4주차 - 나만의 AI 에이전트 만들기
5주차 - 자동화 확장 도구 실전 활용법 (OpenClaw · 헤르메스)
6주차 - 실무를 자동화하는 최종 프로젝트

---

[수강 후 기대효과 - 구체적으로 표현]
- 코딩 없이 AI 도구로 실제 동작하는 웹페이지 제작 가능
- 반복 업무를 처리하는 나만의 AI 에이전트 보유
- 업무 자동화 미니 프로젝트 완성본 1개 이상 보유
- AI 툴 선택·조합 능력 습득으로 업무 생산성 향상

---

[필수 섹션 구성 - 순서 준수]
1. Header - 로고 + 네비게이션 (강의소개 / 커리큘럼 / FAQ) + 수강신청 버튼
2. Hero - 헤드라인 + 서브카피 + CTA 버튼 ("지금 수강 신청하기") + 강의 키워드 뱃지
3. 감정 공감 섹션 - "이런 고민 있으셨나요?" (공감 카드 3~4개)
4. 수강 대상 - "이런 분께 딱 맞습니다" (아이콘 + 설명 카드)
5. 수강 후 기대효과 - 6주 후 달라지는 것 (아이콘 + 짧은 설명)
6. 커리큘럼 - 주차별 인터랙티브 타임라인 (클릭 시 아코디언 상세 펼치기)
7. 강의 특징 - 차별화 포인트 3가지 (실습 중심 / 결과물 중심 / 자동화 중심)
8. 강사 소개 - [강사명] 플레이스홀더
9. FAQ - 자주 묻는 질문 아코디언 (난이도 / 준비물 / 수강 기간 / 환불 등)
10. 최종 CTA - "지금 신청하고 [할인혜택] 받기" + 마감 임박 배지
11. Footer - 이메일 / 카카오채널 / SNS 링크 (플레이스홀더)

---

[인터랙션 & 애니메이션]
- 스크롤 시 각 섹션 페이드인 (Intersection Observer API 사용)
- Hero CTA 버튼 글로우 펄스 효과
- 커리큘럼 주차 클릭 시 아코디언 펼치기/접기
- FAQ 아코디언 열기/닫기
- 카드 호버 시 위로 떠오르는 효과 (transform + box-shadow)
- 모바일 햄버거 메뉴 토글
- 네비게이션 클릭 시 해당 섹션으로 스무스 스크롤
- 수강신청 버튼 클릭 시 신청 폼 모달 팝업

---

[디자인 스타일]
- 다크 테마 기반
- 네온 퍼플 / 블루 그라디언트 강조
- Glassmorphism 카드 UI
- 레퍼런스: Linear.app, Vercel 랜딩 페이지 느낌
- 미래지향적이지만 비전공자도 친근하게 느낄 수 있는 톤

[전환 최적화 요소]
- Hero와 최종 섹션 두 곳에 CTA 버튼 배치
- "마감 임박" / "조기 신청 혜택" 배지 표시 (플레이스홀더)
- 신뢰 요소: 수강생 수, 만족도 수치 (플레이스홀더, 예: "수강생 [N]명 / 만족도 [N]%")
- 가상 수강 후기 카드 2~3개

[기술 스펙]
- HTML + CSS + JavaScript 순수 바닐라만 사용 (외부 라이브러리·CDN 전면 금지)
- 단일 index.html 파일로 완성 (CSS · JS 모두 인라인)
- 모바일 퍼스트 반응형 (320px ~ 1440px 완전 대응)
- 터치 친화적 UI (버튼 최소 48x48px)
- 시맨틱 HTML 태그 사용
- 접근성 준수 (alt 속성, aria-label, 포커스 상태)
- SEO 메타태그 포함 (title, description, og 태그)
- CSS 변수(Custom Properties)로 색상·폰트 일괄 관리
- Lighthouse 성능 최적화 고려 (이미지 없이 CSS 그라디언트·SVG 아이콘으로 대체)
